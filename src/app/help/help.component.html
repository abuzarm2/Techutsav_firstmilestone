Orthogonal Array Testing (OAT)

Orthogonal Array formula is represented as

·         Runs – Number of rows which represents the number of test cases that are required to be performed

·         Factors – Number of columns which represents the number of parameters that can be handled.

·         Levels – Maximum number of values that can be taken on any parameter (Factors)

Example 1:

Consider an application that has a page which takes the input for ordering a mobile phone. Following fields take input from the user and navigates to the payment page.

Model – Samsung, Motorola, Nokia

Storage – 80GB, 100GB, 128GB

Memory – 4GB, 6GB, 8GB

Processors – Qualcomm, Snapdragon, Mediatex

To test all the combinations related to the above 4 parameters, we require 3 x 3 x 3 x 3 = 81 test cases.

Using orthogonal array testing methodology, we require only 9 test cases to test all the unique combinations.

Here, factors are – Model, Storage, Memory and Processor

Levels – 3 levels (each factor has 3 unique value)

The orthogonal array is represented as L9 (34)

Let us build an orthogonal array for 4 factor and 3 levels.

Here, Factor 1, Factor 2, Factor 3 and Factor 4 are replaced by Model, Storage, Memory and Processor and 1 and 2 are replaced by their respective value under each factor.
